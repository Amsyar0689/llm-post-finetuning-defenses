| Script                         | Purpose                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| ------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **prepare_data.py**            | Downloads and preprocesses the datasets used in the paper (Alpaca for instruction‑following, AdvBench for harmful prompts, BeaverTails for safety and SST‑2/MT‑Bench for utility).  Required flags include `--output_dir` for the processed dataset root and `--model_name` to specify the base model whose tokenizer defines sequence length.  By default it constructs ratio‑specific subdirectories `ratio_1`, `ratio_5` and `ratio_10`. |
| **train_attacks.py**           | Performs QLoRA fine‑tuning of the base model on harmful data to create a compromised adapter.  Key arguments: `--learning_rate`, `--epochs`, `--ratio` (harmful data proportion), `--dataset_root`, `--output_root`, `--base_model`, `--max_seq_length`, `--per_device_batch_size`, `--gradient_accumulation_steps`, `--gradient_checkpointing` and `--seed`.                                                                               |
| **train_vaccine.py**           | Implements the Vaccine defense baseline.  It trains a robust adapter by perturbing harmful samples under an ℓ₁/ℓ₂ budget.  Important flags: `--dataset_path`, `--ratio`, `--learning_rate`, `--epochs`, `--vaccine_rho` (perturbation budget), `--base_model`, `--max_seq_length`, `--per_device_batch_size`, `--gradient_accumulation_steps`, `--safety_anchor_samples` and `--output_root`.                                               |
| **apply_antidote.py**          | Applies the Antidote defense by performing weight pruning (Wanda) on a compromised adapter.  It takes `--model_path` for the base model, `--adapter_path` for the harmful adapter, `--output_dir` where the pruned weights are written, and optional `--prune_ratio`, `--calibration_dataset`, `--calibration_samples`, `--batch_size` and `--max_seq_length`.                                                                              |
| **evaluate_attack_success.py** | Measures the Attack Success Rate (ASR) by running the harmful test prompts through the defended model.  Arguments: `--base_model`, `--adapter_path`, `--judge_model` (safety classifier) and `--output_file` for writing metrics.                                                                                                                                                                                                           |
| **evaluate_unified.py**        | Computes both safety and utility metrics on BeaverTails, SST‑2 and MT‑Bench in a single run.  Use `--base_model`, `--adapter_path`, `--judge_model` and `--output_file` to control the evaluation.                                                                                                                                                                                                                                          |
| **safety_utility_eval.py**     | Aggregates multiple evaluation JSON files into a tabulated CSV and computes summary statistics.  Use `--results_dir` to specify where individual evaluation outputs live.                                                                                                                                                                                                                                                                   |
| **plot_results.py**            | Generates simple matplotlib figures from aggregated results.  Required flags: `--results_dir` and `--output_dir` for saving PNG files.                                                                                                                                                                                                                                                                                                      |
| **run_pipeline.sh**            | Original end‑to‑end driver that creates datasets, performs a grid of harmful ratios/learning rates/epochs via `train_attacks.py`, and optionally runs evaluation and Antidote pruning.  It accepts many environment variables (e.g., `RATIOS`, `LEARNING_RATES`, `EPOCHS_LIST`, `RUN_EVAL`, `RUN_ANTIDOTE`, `EVAL_ADAPTER_PATH`) to customise behaviour.                                                                                    |
| **run_vaccine_tiers.sh**       | Convenience wrapper for the Vaccine baseline.  It defines three tier presets (weak, moderate, aggressive) with different ratios and learning rates, and sequentially calls `train_vaccine.py` on each.  Environment variables such as `GPU_IDS`, `TIERS`, `VACCINE_RHO`, `PER_DEVICE_BATCH`, `GRAD_ACCUM_STEPS` and `MAX_SEQ_LENGTH` control GPU allocation and hyper‑parameters.                                                           |
| **run_pipeline_tmux.sh**       | Simple wrapper that launches `run_pipeline.sh` inside a detached tmux session for long‑running training on remote machines.                                                                                                                                                                                                                                                                                                                 |
| **run_experiment.sh**          |  Unified experiment driver that supersedes the older shell wrappers.  It parses command‑line flags to select the defense method (none, Antidote, Vaccine, LiSA), loops over user‑supplied hyper‑parameters, calls the appropriate training scripts, applies the chosen defense and runs evaluation.  See `run_experiment.sh --help` or the main README for examples and details.                                                   |
